## Welcome Contributors!

ProjetosVoluntariosTech is a beginner-friendly volunteer tech community that creates websites and applications for churches, non-profits, and small businesses in need. We welcome contributors of all skill levels!

<Note>
  **First-time contributor?** Don't worry! This project is designed to be accessible for developers at any stage of their journey. We're here to help you learn and grow.
</Note>

## Getting Started

<Steps>
  <Step title="Fork the Repository">
    Navigate to the [GitHub repository](https://github.com/Yudikom/ProjetosVoluntariosTech) and click the "Fork" button in the top right corner.
    
    ```bash
    # Clone your forked repository
    git clone https://github.com/YOUR_USERNAME/ProjetosVoluntariosTech.git
    cd ProjetosVoluntariosTech
    ```
  </Step>

  <Step title="Explore the Projects">
    Browse the `source/` directory to see existing projects:
    
    - `projeto1-Escola-Biblica/` - Biblical School portal
    - `projeto1-solucao-refrigeracao/` - Refrigeration business landing page
    
    Each project contains HTML, CSS (inline or external), and JavaScript files.
  </Step>

  <Step title="Choose or Request a Task">
    - Check open [Issues](https://github.com/Yudikom/ProjetosVoluntariosTech/issues) for available tasks
    - Look for issues labeled `good first issue` or `help wanted`
    - Or propose a new project by creating an issue with the `new-project` label
  </Step>

  <Step title="Create a Branch">
    Always create a new branch for your work:
    
    ```bash
    # For new features
    git checkout -b feature/your-feature-name
    
    # For bug fixes
    git checkout -b fix/bug-description
    
    # For new projects
    git checkout -b project/project-name
    ```
  </Step>

  <Step title="Make Your Changes">
    Follow our [code guidelines](/contributing/code-guidelines) and make your changes. Test thoroughly before committing.
    
    <Tip>
      Open the HTML files directly in your browser to preview changes. No build process required for basic projects!
    </Tip>
  </Step>

  <Step title="Commit Your Work">
    Write clear, descriptive commit messages:
    
    ```bash
    # Stage your changes
    git add .
    
    # Commit with a descriptive message
    git commit -m "Add responsive navigation to Escola Biblica project"
    
    # Push to your fork
    git push origin feature/your-feature-name
    ```
    
    <Warning>
      Always review your changes before committing. Make sure you haven't accidentally included sensitive data or personal information.
    </Warning>
  </Step>

  <Step title="Submit a Pull Request">
    Go to the original repository and click "New Pull Request". Fill out the PR template with:
    
    - Clear description of what you changed
    - Screenshots (if UI changes)
    - Reference to related issue(s)
    - Testing steps
    
    Our maintainers will review your PR and provide feedback!
  </Step>
</Steps>

## Contributing Code

When contributing code to existing projects or creating new ones:

<Accordion title="What Makes a Good Contribution">
  - **Clean, semantic HTML** - Use appropriate tags for accessibility
  - **Responsive design** - Works on mobile, tablet, and desktop
  - **Cross-browser compatible** - Test on Chrome, Firefox, and Safari
  - **Performance optimized** - Minimize external dependencies
  - **Well-commented code** - Help others understand your work
  - **Follows project patterns** - Match the existing code style
</Accordion>

<Accordion title="Project Types We Accept">
  - Church and ministry websites
  - Non-profit organization pages
  - Small business landing pages
  - Community service applications
  - Educational portals
  - Charity campaign sites
</Accordion>

## Contributing Documentation

Help improve our documentation:

- Fix typos and grammatical errors
- Add missing information or clarifications
- Create tutorials and how-to guides
- Translate documentation to other languages
- Improve code examples

```bash
# Documentation files are in the docs/ directory
cd docs/

# Edit existing .mdx files or create new ones
vim contributing/new-guide.mdx
```

## Contributing Design

Designers can contribute by:

- Creating UI mockups for new projects
- Improving color schemes and typography
- Ensuring WCAG accessibility compliance
- Designing logos and branding materials
- Creating reusable component libraries

<Tip>
  Share your designs as Figma files, Adobe XD links, or detailed screenshots in the project issues.
</Tip>

## Contributing Testing

Quality assurance is crucial:

1. Test new features on multiple devices
2. Report bugs with detailed reproduction steps
3. Verify accessibility with screen readers
4. Check performance on slow connections
5. Validate HTML/CSS using W3C validators

## Code of Conduct

<Note>
  We are committed to providing a welcoming and inclusive environment. Please treat all community members with respect and kindness.
</Note>

**Expected Behavior:**
- Be respectful and constructive in feedback
- Welcome newcomers and help them learn
- Focus on what's best for the community
- Show empathy towards other community members

**Unacceptable Behavior:**
- Harassment or discriminatory language
- Personal attacks or trolling
- Publishing others' private information
- Spam or promotional content

## Recognition

All contributors will be:
- Added to our CONTRIBUTORS.md file
- Credited in project documentation
- Recognized in release notes
- Building a portfolio of real-world projects

<Tip>
  Use these volunteer projects as portfolio pieces when applying for jobs. Real-world experience matters!
</Tip>

3. Open a new issue with the `question` label
